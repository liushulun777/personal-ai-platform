## 2. 表结构设计

### 2.1 biz_article_chunk 表

```sql
CREATE TABLE biz_article_chunk (
    id              BIGINT          PRIMARY KEY,
    article_id      BIGINT          NOT NULL,
    chunk_index     INT             NOT NULL,
    content         TEXT            NOT NULL,
    embedding       vector(768),
    author_id       BIGINT          NOT NULL,
    category_id     BIGINT,
    tags            VARCHAR(500),
    publish_time    TIMESTAMP,
    create_time     TIMESTAMP       NOT NULL DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_article_chunk_article ON biz_article_chunk(article_id);
CREATE INDEX idx_article_chunk_author ON biz_article_chunk(author_id);
CREATE INDEX idx_article_chunk_category ON biz_article_chunk(category_id);
CREATE INDEX idx_article_chunk_publish_time ON biz_article_chunk(publish_time);
```

**设计要点：**
- `embedding vector(768)`：使用 pgvector 存储 768 维向量（与 nomic-embed-text 模型匹配）
- 冗余字段 `author_id`、`category_id`、`tags`、`publish_time`：避免跨表联查，直接在向量表内完成多维筛选
- `tags` 使用 VARCHAR(500) 存储 JSON 数组，如 `["Java","Spring"]`
- 不使用逻辑删除（追加型表，通过删除重建实现更新）

### 2.2 前端可配字段

| 字段 | 类型 | 说明 | 筛选支持 |
|------|------|------|----------|
| author_id | Long | 作者ID | ✅ 精确匹配 |
| category_id | Long | 分类ID | ✅ 精确匹配 |
| tags | JSON | 标签列表 | ✅ 包含匹配 |
| publish_time | Timestamp | 发布时间 | ✅ 日期范围 |
| content | Text | 分块内容 | ✅ 语义搜索 |

## 3. 架构设计

### 3.1 数据流向

```
blog-service (port 1083)
    │
    │ Kafka: article-publish / article-update / article-delete
    │ (事务提交后发送)
    ▼
search-service (port 1085)
    │
    ├──► ElasticSearch (articles 索引)
    │    └─ 全文搜索、关键词匹配
    │
    └──► PostgreSQL + pgvector (biz_article_chunk 表)
         └─ 向量语义搜索、多维筛选
```

### 3.2 搜索模式

| 模式 | 接口 | 说明 |
|------|------|------|
| 全文搜索 | POST /api/search/articles | ES 关键词匹配 + 多维筛选 |
| 语义搜索 | POST /api/search/articles/semantic | pgvector 向量相似度 + 多维筛选 |

### 3.3 语义搜索流程

```
用户输入查询文本
    │
    ▼
EmbeddingModel.embed(queryText) → 查询向量
    │
    ▼
pgvector 余弦相似度搜索
  WHERE author_id = ? AND category_id = ? AND publish_time BETWEEN ? AND ?
  ORDER BY embedding <=> query_vector
  LIMIT topK
    │
    ▼
按 article_id 聚合，取最高相似度
    │
    ▼
从 ES 获取完整文档（带高亮）
    │
    ▼
按语义相似度排序返回
```

### 4.3 核心代码片段

#### ArticleChunkMapper - 向量搜索 SQL

```java
@Select("<script>" +
        "SELECT c.id, c.article_id, c.chunk_index, c.content, c.author_id, " +
        "c.category_id, c.tags, c.publish_time, c.create_time, " +
        "(1 - (c.embedding <![CDATA[<=>]]> #{embeddingStr}::vector)) as similarity " +
        "FROM biz_article_chunk c " +
        "<where>" +
        "  <if test='authorId != null'>AND c.author_id = #{authorId}</if>" +
        "  <if test='categoryId != null'>AND c.category_id = #{categoryId}</if>" +
        "  <if test='startDate != null'>AND c.publish_time <![CDATA[>=]]> #{startDate}</if>" +
        "  <if test='endDate != null'>AND c.publish_time <![CDATA[<=]]> #{endDate}</if>" +
        "</where>" +
        "ORDER BY c.embedding <![CDATA[<=>]]> #{embeddingStr}::vector " +
        "LIMIT #{limit}" +
        "</script>")
List<ArticleChunk> searchSimilar(@Param("embeddingStr") String embeddingStr,
                                 @Param("authorId") Long authorId,
                                 @Param("categoryId") Long categoryId,
                                 @Param("startDate") LocalDateTime startDate,
                                 @Param("endDate") LocalDateTime endDate,
                                 @Param("limit") Integer limit);
```

#### ArticleChunkServiceImpl - 文章分块处理

```java
@Override
public void processArticle(ArticleDocument document) {
    // 1. 删除旧分块
    deleteByArticleId(document.getId());

    // 2. 拼接标题+摘要+内容
    String fullText = buildFullText(document);

    // 3. 分块
    List<String> chunks = chunkingService.chunk(fullText, chunkSize, chunkOverlap);

    // 4. 批量生成向量嵌入
    List<float[]> embeddings = embeddingService.embedBatch(chunks);

    // 5. 保存分块（含冗余字段）
    for (int i = 0; i < chunks.size(); i++) {
        ArticleChunk chunk = new ArticleChunk();
        chunk.setArticleId(document.getId());
        chunk.setChunkIndex(i);
        chunk.setContent(chunks.get(i));
        chunk.setEmbedding(embeddings.get(i));
        chunk.setAuthorId(document.getAuthorId());
        chunk.setCategoryId(document.getCategoryId());
        chunk.setTags(tagsJson);
        chunk.setPublishTime(document.getPublishTime());
        articleChunkMapper.insert(chunk);
    }
}
```

#### ArticleEventListener - 事件处理

```java
@KafkaListener(topics = "article-publish", ...)
public void handleArticlePublish(ArticleDocument document, ...) {
    // 1. 同步到 ES
    searchService.indexArticle(document);
    // 2. 分块 + 向量嵌入 + 存储到 pgvector
    articleChunkService.processArticle(document);
}
```

## 8. 注意事项

1. **向量维度**：nomic-embed-text 模型输出 768 维向量，与表定义 `vector(768)` 一致
2. **分块大小**：默认 500 字符，重叠 50 字符，可通过配置调整
3. **更新策略**：文章更新时先删除旧分块再创建新分块，保证数据一致性
4. **性能考虑**：向量搜索在大数据量时建议使用 HNSW 索引（当前使用 IVFFlat）
5. **错误处理**：分块/嵌入失败不影响文章发布主流程，仅记录日志
