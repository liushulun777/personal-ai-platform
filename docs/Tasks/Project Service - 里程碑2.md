# Project Service - 里程碑2：AI驱动开发系统

> **核心目标**：让 AI 能真正"控制" project-service，最终实现 AI 自动完成开发工作。

---

## 🎯 总体路线图

```
第一阶段：任务系统闭环验证
    ↓
第二阶段：AI开始"做真实开发任务"
    ↓
第三阶段：AI + MCP 执行真实系统操作
    ↓
第四阶段：AI自动完成一个完整功能（关键里程碑）
    ↓
第五阶段：AI驱动开发系统（高级）
```

---

## ✅ 第一阶段：任务系统闭环验证

### 🎯 目标

验证 AI 能不能真正"控制 project-service"

### 📌 AI 要执行的任务

#### 1. 自动创建测试项目

**执行动作**：调用 project-service 创建 pm_project

**创建项目**：`微服务融合`

#### 2. AI 自动拆解任务（核心）

**输入**：

```
1、目前项目微服务拆分太多了，启动项目调试的时候。内容系统内存太多了，需要融合一部分。先整理文档，待我审阅后执行。
2、目前的项目有审阅功能吗？需要加一个审阅功能，复杂的任务、项目。bug改动较大时都需要审阅后执行。

```

**AI 输出任务**（写入 pm_task）：


#### 3. AI 自动推进任务状态

**模拟执行状态流转**：

```
BACKLOG → READY → DOING → REVIEW → DONE
```

#### 4. 写执行日志（pm_task_execution）

**AI 必须记录**：
- 做了什么
- 调用了什么工具
- 输出结果

### 🎯 成功标准

- [x] task 可以自动生成
- [x] status 可以自动流转
- [x] execution 可以记录

## 🤖 第二阶段：AI开始"做真实开发任务"

### 🎯 目标

让 AI 不只是"管理任务"，而是**开始生成真实代码**

### 📌 AI 执行任务

#### 任务1：生成 blog-service 接口

**目标**：生成博客列表接口（分页）

**AI 必须**：
1. 创建 task
2. 生成代码结构

#### 任务2：数据库设计

**目标**：设计博客表 + 标签表 + 分类表

**AI 输出**：
- SQL 建表语句
- Entity 实体类
- Mapper 接口

#### 任务3：接口开发

**目标**：实现 blog/list 接口

**AI 执行**：
1. 写 Controller
2. 写 Service
3. 写 Mapper
4. 写测试请求

### 🎯 成功标准

- [x] AI 能生成结构化任务
- [x] AI 能生成代码（哪怕是伪代码也行）
- [x] AI 能拆解开发步骤

## 🧩 第三阶段：AI + MCP 执行真实系统操作

### 🎯 目标

让 AI 能"动系统"

### 📌 AI 执行任务

#### 1. 查询数据库结构

**MCP 工具**：`database.query`

**任务**：修改 application.yml

#### 4. 调接口测试

**任务**：调用 /blog/list

### 🎯 成功标准

- [x] AI 可以调用工具
- [x] AI 可以操作文件
- [x] AI 可以验证接口

## 🔥 第四阶段：AI自动完成一个完整功能（关键里程碑）

### 🎯 目标

让 AI 独立完成**一个完整功能开发闭环**

### 📌 示例任务

**目标**：实现一个"博客发布 + 列表 + 查询"功能

- 接口设计
- Service
- Controller

- 写 SQL
- 写 Java 代码
- 写 Mapper
- 写接口

#### Step 3：验证

- 调接口
- 修 bug

#### Step 4：更新状态

- 标记为 DONE

- [x] 无需人工拆代码
- [x] 可复现
