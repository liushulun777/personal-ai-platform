# 权限分配页面优化需求文档

## 1. 背景与问题

### 当前问题

当前角色权限分配存在以下操作不便的问题：

1. **TreeSelect 组件拥挤**：菜单权限使用 Naive UI 的 `NTreeSelect` 组件，弹窗仅 500px 宽，树形结构层级深（目录→菜单→按钮），展开后难以操作
2. **无批量操作**：无法一键全选/反选某个目录下的所有权限
3. **权限层级不清晰**：目录、菜单、按钮三种类型混在一起，没有视觉区分
4. **无权限预览**：编辑角色时无法直观看到当前角色拥有的完整权限列表
5. **用户角色分配缺少反馈**：用户管理中分配角色后，无法直观看到用户拥有哪些权限

## 2. 目标

- 权限选择操作直观、高效
- 支持批量选择/取消
- 三种权限类型（目录、菜单、按钮）视觉区分清晰
- 操作步骤最少化

#### 交互规范

| 操作 | 行为 |
|------|------|
| 点击目录节点 | 级联选中/取消该目录下所有子菜单和按钮 |
| 点击菜单节点 | 级联选中/取消该菜单下所有按钮 |
| 点击按钮节点 | 仅影响该按钮 |
| [全选] | 选中所有节点 |
| [全不选] | 取消所有节点 |
| [展开/折叠] | 切换树的展开状态 |
| 搜索框 | 按菜单名称过滤树节点 |

#### 节点样式区分

| 类型 | 图标 | 前缀颜色 |
|------|------|----------|
| 目录 (menuType=0) | 📁 文件夹图标 | 蓝色 |
| 菜单 (menuType=1) | 📄 页面图标 | 绿色 |
| 按钮 (menuType=2) | 🔘 按钮图标 | 橙色 |

## 4. 技术方案

### 4.1 组件选择

- **方案A（推荐）**：使用 Naive UI 的 `NTree` 组件（非 TreeSelect），配合 `checkable` + `cascade` 属性，自定义渲染节点图标和类型标识
- **方案B**：自定义权限树组件，使用 checkbox + 递归渲染

选择方案A的理由：
- `NTree` 比 `NTreeSelect` 有更大的渲染空间
- 支持 `check-strategy="parent"` 级联策略
- 内置搜索、展开/折叠功能
- 样式可通过 `render-label` 自定义

### 4.2 弹窗尺寸

- 角色编辑弹窗：`width: 780px`，最小高度 `520px`
- 用户编辑弹窗：`width: 560px`（当前已优化）

### 4.3 数据流

```
角色编辑:
  1. 打开弹窗 → 加载菜单树 (getMenuTree)
  2. 编辑模式 → 加载角色详情 (getRoleById) → 回显 menuIds
  3. 用户操作树 → 更新 formData.menuIds
  4. 提交 → createRole/updateRole (含 menuIds)

用户编辑:
  1. 打开弹窗 → 加载用户详情 (getUserById) → 获取 roleIds
  2. 加载所有角色 (getAllRoles) → 角色选项列表
  3. 用户选择角色 → 更新 formData.roleIds
  4. 提交 → updateUser + assignRoles
```

## 5. 验收标准

- [ ] 角色编辑弹窗宽度 ≥ 780px，权限树不拥挤
- [ ] 目录/菜单/按钮三种类型有明确的图标和颜色区分
- [ ] 支持一键全选/全不选
- [ ] 支持按名称搜索过滤权限节点
- [ ] 点击目录可级联选中所有子项
- [ ] 用户编辑可查看和分配角色
- [ ] 权限修改后登录用户需重新登录生效
