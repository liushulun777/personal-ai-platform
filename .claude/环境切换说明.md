# 环境切换配置说明

## 配置概述

所有微服务的 `application.yml` 已配置 `INFRA_HOST` 变量，支持在公司和家里环境之间快速切换。

| 环境 | IP 地址 | Profile |
|------|---------|---------|
| 公司 | 172.18.0.34 | `company` |
| 家里 | 192.168.5.2 | `home` |

## 使用方式

### 修改配置文件切换（推荐）

直接修改每个服务的 `application.yml` 中的 `spring.profiles.active` 配置：

```yaml
spring:
  profiles:
    active: home  # 改为 company 即切换到公司环境
```

**当前默认：家里环境 (home)**

### 通过 JVM 参数覆盖（可选）

如果不想修改配置文件，可以通过 JVM 参数覆盖：

```bash
# 公司环境
java -jar -Dspring.profiles.active=company xxx.jar

# 家里环境
java -jar -Dspring.profiles.active=home xxx.jar
```

## 注意事项

- Ollama 服务 (192.168.9.95) 暂不切换，公司和家里相同
- Docker Compose 部署的配置不在此范围内
- 切换环境后需要重启服务生效
